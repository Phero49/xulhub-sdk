Xulhub SDK

Create interactive exercises and custom cell content for educational notebooks

Introduction

The Xulhub SDK enables you to create custom interactive content for educational notebooks. Build quizzes, exercises, visualizations, and more that seamlessly integrate with the Xulhub platform.

Understanding the Notebook Structure

Xulhub notebooks follow a hierarchical structure:

📓 Notebook (The entire document)
  └─ 📄 Cell (A section/block in the notebook)
      └─ 🧩 Cell Content (The interactive component inside a cell)
          ← Your extension runs here
🎯 Extension Scope: Your extensions/plugins operate at the cell content level. Each cell can contain one or more pieces of content, and your extension controls what happens inside that content area. The SDK handles communication between your extension and the parent notebook application.

How It Works

  • Notebook - The top-level container that holds all cells and manages overall state
  • Cells - Individual blocks within a notebook. Each cell can have text, exercises, or custom content
  • Cell Content - Your extension runs here as an isolated component (iframe) within a cell

The SDK provides a bridge between your isolated cell content and the parent application, enabling you to:

  • Save and load content data specific to your cell content
  • Access notebook-wide metadata and settings
  • Control quiz scoring and navigation
  • Communicate with the parent application for features like TTS and dialogs

Key Features

  • Quiz and exercise management with scoring
  • AI-powered answer verification
  • Text-to-speech support for accessibility
  • Auto-generation from various content formats
  • Data persistence and state management
  • Isolated execution environment for security

Installation

Install the SDK in your project:

// Using npm npm install @xulhub/sdk // Using yarn yarn add @xulhub/sdk

Basic Import

import { createNotebookSDK } from '@xulhub/sdk'; const sdk = createNotebookSDK();

Quick Start

Here's a simple example to get you started:

import { createNotebookSDK } from '@xulhub/sdk'; const sdk = createNotebookSDK(); // Wait for SDK to initialize sdk.onReady(() => { // Get existing content data const data = sdk.getContentData(); // Set up quiz verification sdk.quizManager.verifyAnswer = () => { const isCorrect = checkUserAnswer(); return { passed: isCorrect, points: isCorrect ? 10 : 0, next: isCorrect }; }; // Show verification button when ready sdk.quizManager.showVerificationButton(); }); // Handle errors sdk.onError((error) => { console.error('SDK Error:', error); });

Lifecycle & Initialization

onReady(callback, config?)

Registers a callback to execute when the SDK is fully initialized.

onReady(callback: () => void, config?: Config): void
  • callback Function - Function to execute when ready
  • config Config (optional) - SDK configuration options

onError(callback)

Registers an error handler for SDK errors.

onError(callback: (error: Error) => void): void

Initialization Properties

Property Type Description
isInitialized boolean Whether SDK has completed initialization
isPublished boolean Whether notebook is in published/read-only mode
cellIndex number Index of current cell in notebook
contentPosition number Position of content within the cell

Data Management

getContentData()

Retrieves the current cell content data.

getContentData(): T | null
interface QuizData { question: string; options: string[]; correctAnswer: number; } const data = sdk.getContentData<QuizData>(); console.log(data.question);

saveContent(data)

Saves content data to the parent application.

saveContent(data: T): void
Important: Only call saveContent after SDK is initialized (inside onReady callback).
const updatedData = { question: "What is 2+2?", options: ["3", "4", "5"], correctAnswer: 1 }; sdk.saveContent(updatedData);

getNotebookMetadata()

Fetches metadata about the current notebook.

getNotebookMetadata(): Promise
const meta = await sdk.getNotebookMetadata(); console.log(`Notebook: ${meta.title}`); console.log(`Created: ${meta.createdAt}`);

Configuration

Configure SDK behavior using the Config object:

Property Type Default Description

hideCheckButton boolean false Hide default check button
hasAutoGen boolean false Enable auto-generation support
sdk.onReady(() => { // Your initialization code }, { height: "500px", hideCheckButton: true, hasAutoGen: true });

Quiz Manager

The QuizManager handles all quiz-related functionality including answer verification, scoring, and navigation.

Access the QuizManager through sdk.quizManager after SDK initialization.

Properties

Property Type Description
totalScore number Total accumulated score for this quiz

Answer Verification

verifyAnswer

Function that checks if the user's answer is correct. You must implement this function.

verifyAnswer: () => { passed: boolean; points: number; next: boolean; }
// Implement verification logic sdk.quizManager.verifyAnswer = () => { const userAnswer = getUserInput(); const correctAnswer = "Paris"; const isCorrect = userAnswer === correctAnswer; return { passed: isCorrect, points: isCorrect ? 10 : 0, next: isCorrect // Allow progression if correct }; };

submitAnswer(options)

Notifies parent application that user has answered a question.

submitAnswer(options: { passed: boolean; points: number; next: boolean; playSound?: boolean; }): void
  • passed boolean - Whether answer was correct
  • points number - Points awarded
  • next boolean - Whether user can proceed
  • playSound boolean (optional) - Play feedback sound

AI-Powered Verification

verifyWithAi(inputData, prompt, output)

Verifies user input using AI with a custom prompt and expected output format.

verifyWithAi(inputData: any, prompt: string, output: T): Promise

Perfect for open-ended questions, essay grading, code review, or complex answers requiring contextual understanding.

Examples

Grade an Essay

const result = await sdk.quizManager.verifyWithAi( userEssay, "Grade this essay on clarity, grammar, and argument strength. Provide scores 0-10.", { clarity: 0, grammar: 0, argument: 0, feedback: "" } ); console.log(`Clarity: ${result.clarity}/10`); console.log(`Feedback: ${result.feedback}`);

Verify Code Solution

const result = await sdk.quizManager.verifyWithAi( userCode, "Check if this code correctly implements bubble sort. Identify bugs if any.", { correct: false, bugs: [], suggestions: "" } ); if (!result.correct) { showBugs(result.bugs); }

Check Mathematical Reasoning

const result = await sdk.quizManager.verifyWithAi( userExplanation, "Evaluate if the student correctly explained the Pythagorean theorem.", { understands_concept: false, score: 0, missing_points: [] } );

Text-to-Speech (TTS)

Add audio accessibility to your content with built-in text-to-speech functionality.

TTS Markup Format

Format: ▶️[lang_code][hide] text to read ⏸️
  • ▶️ - TTS start marker
  • ⏸️ - TTS stop marker
  • [lang_code] - Language code (e.g., "en", "es", "fr", "ja")
  • [hide] - Optional: hides text but still reads it aloud

processTTS(element, size)

Converts TTS markup into interactive play/pause buttons.

processTTS(element: HTMLElement, size: string): Promise
const container = document.getElementById('content'); await sdk.processTTS(container, "24px");

reverseTTS(innerHTML)

Converts TTS buttons back to original markup format.

reverseTTS(innerHTML: string): Promise
const originalMarkup = await sdk.reverseTTS(element.innerHTML);

Example Usage

// HTML content with TTS markup const content = ` <p>▶️[en] Hello, welcome to the lesson! ⏸️</p> <p>▶️[es] Hola, bienvenidos a la lección! ⏸️</p> <p>▶️[en][hide] This text is hidden but will be read ⏸️</p> `; document.getElementById('content').innerHTML = content; // Process TTS markup await sdk.processTTS( document.getElementById('content'), "24px" );

Auto-Generation

Enable your extension to automatically generate cells from imported content like HTML, Markdown, or JSON.

Note: Set hasAutoGen: true in your config to enable this feature.

Content Generator Configuration

sdk.contentGenerator = { contentType: 'htmlElement', // 'htmlElement' | 'markdown' | 'json' instructionFormat: 'Generate a multiple-choice question with 4 options', processImport: (input) => { // Transform input into cell content return [{ cellType: 'exercise', cellContent: parseQuestion(input), text: 'Auto-generated question' }]; } };

ProcessImport Function

processImport(input)

Transforms preprocessed content into structured cell data.

processImport(input: string | HTMLElement): ProcessImportOutput[] | null

Input Types

  • HTML - Host app parses pasted HTML into an HTMLElement
  • Markdown - Raw string sent as-is
  • JSON - Object sent directly

Output Format

interface ProcessImportOutput { cellType: 'exercise' | 'note'; cellContent: any; // Your extension's data format text: string | null; // Initial text/prompt to display }

Complete Example

const sdk = createNotebookSDK(); sdk.contentGenerator = { contentType: 'htmlElement', instructionFormat: 'Create quiz questions from this content', processImport: (input) => { if (input instanceof HTMLElement) { // Extract questions from HTML const questions = input.querySelectorAll('.question'); return Array.from(questions).map(q => ({ cellType: 'exercise', cellContent: { question: q.querySelector('.text').textContent, options: Array.from( q.querySelectorAll('.option') ).map(o => o.textContent), correctAnswer: parseInt( q.getAttribute('data-correct') ) }, text: q.querySelector('.text').textContent })); } return null; } }; sdk.onReady(() => { console.log('Auto-generation enabled'); }, { hasAutoGen: true });

Complete Examples

Multiple Choice Quiz

import { createNotebookSDK } from '@xulhub/sdk'; const sdk = createNotebookSDK(); sdk.onReady(() => { const data = sdk.getContentData(); // Render quiz UI renderQuiz(data); // Setup verification sdk.quizManager.verifyAnswer = () => { const selected = document.querySelector('input[name="answer"]:checked'); const isCorrect = selected && parseInt(selected.value) === data.correctAnswer; return { passed: isCorrect, points: isCorrect ? 10 : 0, next: isCorrect }; }; // Show verification button sdk.quizManager.showVerificationButton(); }, { height: '400px' }); function renderQuiz(data) { const container = document.getElementById('quiz'); container.innerHTML = ` <h3>${data.question}</h3> ${data.options.map((opt, i) => ` <label> <input type="radio" name="answer" value="${i}"> ${opt} </label> `).join('')} `; }

AI-Verified Essay Question

const sdk = createNotebookSDK(); sdk.onReady(() => { const checkButton = document.getElementById('check'); checkButton.addEventListener('click', async () => { const essay = document.getElementById('essay').value; try { const result = await sdk.quizManager.verifyWithAi( essay, `Evaluate this essay about climate change. Check for: understanding of concepts, argument quality, use of evidence, and clarity of writing.`, { understanding: 0, argument: 0, evidence: 0, clarity: 0, feedback: "", totalScore: 0 } ); // Display results showFeedback(result); // Submit answer sdk.quizManager.submitAnswer({ passed: result.totalScore >= 70, points: result.totalScore, next: result.totalScore >= 70, playSound: true }); } catch (error) { console.error('Verification failed:', error); } }); }, { hideCheckButton: true });

Best Practices

✅ Do's

  • Always wait for onReady before accessing SDK methods
  • Handle errors gracefully with onError callbacks
  • Type your data with TypeScript generics for better IDE support
  • Save content regularly to prevent data loss
  • Use AI verification for complex, open-ended questions
  • Implement showCorrectAnswer for better learning experience
  • Add TTS support for accessibility

❌ Don'ts

  • Don't call SDK methods before initialization
  • Don't save null or undefined data
  • Don't ignore the published state (disable editing when published)
  • Don't make assumptions about cell position or index
  • Don't block the UI during AI verification (show loading states)

Performance Tips

  • Debounce save operations to avoid excessive calls
  • Cache processed TTS elements when possible
  • Use efficient DOM manipulation techniques
  • Clean up event listeners when appropriate
  • Optimize AI prompts to be concise and clear

Troubleshooting

Common Issues

SDK not initializing

Problem: SDK times out or doesn't respond

Solution:

// Check if running in iframe if (window.parent === window) { console.error('SDK must run in an iframe'); } // Add error handler sdk.onError((error) => { console.error('SDK Error:', error); // Show user-friendly message });

Data not persisting

Problem: Content changes aren't saved

Solution:

// Ensure SDK is initialized if (!sdk.isInitialized) { console.error('Wait for onReady before saving'); return; } // Validate data before saving if (data && typeof data === 'object') { sdk.saveContent(data); }

TTS not working

Problem: TTS buttons don't appear or don't speak

Solution:

// Ensure proper markup format const text = '▶️[en] Hello world ⏸️'; // Correct const wrong = '▶️ Hello world ⏸️'; // Missing [lang] // Wait for processing to complete await sdk.processTTS(element, "24px"); console.log('TTS processing complete');
